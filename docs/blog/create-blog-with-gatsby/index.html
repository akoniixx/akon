บันทึกการสร้างเว็บบล็อกด้วย Gatsby

October 9th, 2019

สวัสดีครับ นี่เป็นการกลับมาเขียนบล็อคในรอบ 5-6 ปีได้ สมัย blogger.com ยังฮิตๆ ด้วยความคันมือบวกกับตอนนี้ได้ทำงานเป็น web developer แบบจริงๆจังๆ เลยต้องหาความรู้อยู่เรื่อยๆ ซึ่งผมเป็นคนที่เขียน react อยู่แล้วและด้วยความขี้เกียจเขียนเอง เลยมักจะหา library มาทดลองใช้ประจำ 555 จนได้ไปเจอกับเจ้า Gatsby

What are Gatsby?

"Gatsby is a free and open source framework based on React that helps developers build blazing fast websites and apps" พูดง่ายๆมันก็คือ static site generator นั่นเอง ไว้จะเขียนเจาะลึกอีกที เพราะส่วนตัวผมเองก็พึ่งได้เริ่มหัดใช้เเหมือนกัน

Language and tools

  • __Gatsby __ ตอนแรกเลยผมคิดว่าจะใช้ React นี่แหล่ะ แต่ส่วนตัวยังไม่เคยทำ back end แบบเป็นจริงเป็นจังมาก่อน จนได้มาเจอกับเจ้านี่ที่สามารถรองรับ Markdown ได้เลย (ตอนนั้นเลยคิดว่าเอาเจ้านี่แหล่ะ จะได้ไม่ต้องทำหลังบ้านให้ยุ่งยาก5555) แน่นอนว่าเราใช้เจ้านี่ในการ development แล้วก็ build เป็น static file อัพขั้น sever ทั่วไปได้เลย
  • GraphQL เมื่อเริ่มได้ศึกษาเจ้า Gatsby ไปสักระยะ พอพูดถึง Gatsby ก็ต้องนึกถึง Graphql ทันที เรียกว่าถ้าอยากใช้เจ้า Gatsby ให้ทรงพลังที่สุด เราต้องมาเรียนรู้เจ้านี่อย่างเลี่ยงไม่ได้แน่ๆ
  • Contentful เมื่อคิดว่ายังไงๆก็หนี GraphQL ไม่พ้น งั้นเอามันมาใช้เลยก็ได้ฟร่ะ จากตอนแรกที่จะเขียนแค่ Markdown แท้ๆ หลังจากนั้นก็พยายามหาเว็บหรือวีดีโอสอน จนไปเจอคลิปหนึ่งที่เขาสอนใช้ Gatsby กับ Contentful เลยเลือกเจ้านี่มาใช้ในส่วนของการเขียนบล็อก(มีแหล่งให้ลอกโค๊ด สบายใจละ)
  • Github Pages โฮสของเว็บนะครับ หลักๆก็เพราะมันฟรี!!!

Get Started

ผมใช้ Gatsby Starter เป็นตัวช่วยในการ setup และผมเลือกใช้ gatsby_starter เพราะมีตัวอย่างการใช้ Graphql ต่อกับ Contentful ให้ดู ส่วนโครงสร้างจะเป็นดังนี้

  • public เป็น Folder ไว้เก็บ content ต่างๆของเว็บไซ์ เช่น เนื้อหาของบทความ โดยดึงมาอยู่ในรูปแบบของ json
  • src ไว้เก็บตัว source code หลักๆของเว็บ ซึ่งส่วนที่เป็น react.js จะอยู่ในนี้ -component component ต่างๆ จะเก็บไว้ในนี้ -pages หน้าต่างๆนอกเหนือจากตัวบทความ -templates ไว้เก็บตัว templates generate หน้าบทความนั่นเอง
  • gatsbys-config.js เป็นส่วนการตั้งค่าเบี้องต้นของเว็บ และจัดการกับ plugin ต่างๆ
  • gatsby-node.js เป็นส่วนของการเขียน function generate ต่างๆ ในที่นี้ผมใช้ในการดึง graphql ที่ไปอ่าน Contentful มาสร้างเป็นหน้าบทความ

Conclusion

บทความนี้ผมจะยังไม่เจาะลึกถึงรายละเอียดหรื่อเชิงเทคนิคเท่าไรนะครับ อยากให้ผู้ที่สนใจได้ผู้เห็นภาพคร่าวๆว่าต้องเจอกับอะไรบ้าง ซึ่งผมชอบเจ้านี่เอามากๆ เรื่อง Performance จากการทดสอบด้วย Light house เจ้านี่ทำได้ดีมาก พร้อมกับมี plugin อีกมากมาย เขียนโค๊ดไม่กี่บรรทัดเว็บของเราก็พร้อมที่จะ deploy แล้ว ใครที่สนใจเจ้านี่อยู่รับรองว่าไม่ผิดหวังครับ
