สิ่งที่ผมอยากเรียนรู้ให้เร็วที่สุด

November 5th, 2019

ด้วยความที่ตอนนี้ผมทำงานเป็น Front end ที่ทำหน้าที่กึ่ง coding กึ่ง designing เพราะที่บริษัทไม่มี ux/ui designer พี่ที่บริษัทแต่ล่ะคนสกิลการ coding มากกว่าเด็กใหม่อย่างผม เขาเลยโยนส่วนนี้ให้ผมเป็นคนรับผิดชอบ (แต่จริงๆก็เข้าทางผมนะ เพราะสมัยเรียนก็ชอบเรื่องการออกแบบ ux/ui อยู่แล้ว) โดยใช้ react เป็นหลัก ดังนั้นหน้าที่หลักๆของผมเลยก็คือการออกแบบ ux/ui จากนั้นก็ coding ขึ้นโครงโปรเจค เขียน Component ต่างๆ จัด Layout ทำ Responsive แล้วพี่ๆจะเป็นคนจัดการเขียนฟังก์ชั้น เขียน api เขียน redux เอง ทำให้ผมยังไม่มีโอกาสทำในส่วนของ Logic แบบจริงๆจังๆ เลยถือโอกาสนี้มาลิสต์หัวข้อต่างๆที่ต้องพยายามเรียนรู้ให้เร็วที่สุด

Things I want to learn as soon as possible

-Redux: แน่นอนถ้าเขียน React แล้วไม่ใช้เจ้านี่ พอโปรเจคใหญ่ขึ้นมากๆต้องลำบากแน่นอน จริงๆก็ลองเขียนเล่นเองมาบ้างแล้ว แต่คิดว่าถ้าเป็นโปรเจคใหญ่ผมยังคงทำงานจริงไม่ได้แน่ -Node.js: ตอนแรกที่หัดเขียน React เลยต้องกลับไปปูพื้นฐาน javascript กับ jsx ด้วยความที่อยากลองทำงานเป็น back end บ้าง (ถึงจะเคยทำโปรเจคด้วย php laravel ก็เถอะ แต่ลืมไปหมดแล้ว 555)เลยเลือก node.js นี่แหล่ะ จะได้ไม่ต้องกลับไปทวนหลายภาษา -Kotlin: สมัยเรียนสิ่งที่อยากทำที่สุดเลยคือ mobile app เพราะตอนนั้นเป็นอะไรที่บูมมาก เลยได้ลงเรียนวิชา Android ที่เขียนด้วย java ซึ่งผมเป็นคนที่รู้สึกไม่ค่อยถูกกับ java เท่าไร แต่ในที่สุดก็เรียนผ่านมาได้ จนได้มาเจอกับ Kotlin ทำให้ไฟในการเขียน Andriod Native กลับมาอีกครั้ง -Swift: ได้คุยกับเพื่อนที่เขียน ios มันมาขิงว่า ios simulator มันลื่นแล้วก็กินสเปคน้อยกว่า andriod simulator มีความฟินเวลาทำงานมากกว่า ผมก็เลยอยากลองย้ายไปเขียน native ฟัง ios เหมือนกัน -Firebase: มีโอกาสได้ฟังงาน Firebase summit เลยคิดว่าเจ้านี่มันน่าสนใจมากๆ ส่วนผมเคยใช้แค่ส่วน hosting เท่านั้นเอง -Docker: เคยอ่านผ่านๆ แต่ถ้าอยากทำ back end ต้องมาหัดใช้แน่ๆ -Data structure: ตอนสมัยเรียนเป็นวิชาที่โหดมากๆ เพื่อนร่วมชั้นตกกันไปหลายคน ตัวผมเองก็เกือบไม่รอด แถมตอนนี้รู้สึกว่าความรู้ที่ได้มาก็คืนอาจารย์ไปหมดแล้ว ต้องกลับมาทบทวนด่วนๆ -Automate test: ลืมบอกไป นอกจากผมจะดีไซน์ โค๊ดดิ้ง แล้วผมยังต้องเป็น tester ด้วย เพราะที่บริษัทไม่มี ตอนนั้นก็ทำเป็นแค่ Manual test อย่างเดียว -Secure Coding: ยอมรับเลยว่าหลายๆอย่างที่ใช้งานอยู่ ผมมเองยังไม่รู้เลยมันทำงานยังไง 555 และคิดว่านี่ก็เป็นสิ่งสำคัญที่หลายๆคนอาจลืมเพราะเห็นว่ามันใช้ได้แล้วก็โอเค ปล่อยผ่านเลย -Algorithm: พอได้มาเจอเหล่าเทพทั้งหลาย ถึงได้รู้ตัวเองว่าเราอ่อนหัดเรื่องนี้นัก ถึงแม้จะแก้ปัญหาได้ แต่ Perfomance ต่างกันอย่างสิ้งเชิง

พอมาไล่ๆลิสต์ดู หลายๆอย่างก็ลืมไปแล้วว่าเราจำเป็นต้องเรียนรู้นะ ที่นี้จะได้เริ่มทำสักที
